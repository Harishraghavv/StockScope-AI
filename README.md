# StockScope AI

AI-powered stock research platform for **educational analysis only** — never
buy/sell advice.

> **Complete StockScope AI Project.** This repository contains the fully built StockScope AI platform. It includes Auth, Dashboard, Stock Screener, Company Profiles, User Settings, and a secure Admin Panel. All modules are seamlessly integrated and ready for production deployment with real APIs (Neon Postgres, Twelve Data, Anthropic).

---

## Tech stack (this module)

- **Framework:** Next.js 14 (App Router), TypeScript
- **Styling:** Tailwind CSS
- **Database:** PostgreSQL + Prisma ORM
- **Auth:** JWT (via `jose`) in an httpOnly cookie, bcrypt password hashing
- **Validation:** Zod
- **State:** Zustand (client-side auth/toast state)

## Getting started

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment variables

```bash
cp .env.example .env
```

Then edit `.env`:

- `DATABASE_URL` — point at your PostgreSQL instance
- `JWT_SECRET` — generate one with `openssl rand -base64 48`
- Market data / news / AI keys are placeholders for later modules — leave
  them blank for now, the auth module doesn't use them

**Never commit `.env`.** It's already in `.gitignore`.

### 3. Set up the database

```bash
npm run prisma:generate
npm run prisma:migrate    # creates the users/sessions/audit_logs tables
npm run db:seed           # creates demo@stockscope.ai / admin@stockscope.ai (password: Demo1234)
```

### 4. Run the dev server

```bash
npm run dev
```

Visit `http://localhost:3000`.

## What's implemented

| Feature | Details |
|---|---|
| Register | `/register` — name, email, password with strength rules |
| Login | `/login` — rate-limited (8 attempts / 15 min / IP) |
| Forgot password | `/forgot-password` — generates a reset token (email sending is stubbed; wired up in a later module) |
| Sessions | Signed JWT in an httpOnly, `sameSite=lax` cookie, 7-day expiry |
| Route protection | `middleware.ts` guards `/dashboard`, `/watchlist`, `/portfolio`, `/profile`, `/settings`, `/admin` |
| Role-based access | `/admin/*` requires `role: ADMIN` |
| Audit logging | Registration and login events recorded in `audit_logs` |

| Feature | Details |
|---|---|
| Market indices | NIFTY 50, SENSEX, NIFTY BANK, NIFTY IT — illustrative values |

real API calls (Alpha Vantage / Twelve Data, keyed via `.env`) — no
component code needs to change.

## Stock screener module (mock data)

| Feature | Details |
|---|---|
| Filters | Market cap, sector, P/E, ROE, ROCE, EPS, revenue/profit growth, debt/equity, dividend yield, RSI, 52-week high/low proximity |
| Sorting | Click any column header to sort; click again to reverse |
| Search | Filter by symbol or company name |
| Export CSV | Downloads current filtered results as a `.csv` file |
| Save screener | Name and save a filter combination — persisted per-user in Postgres (`saved_screeners` table) |
| Load / delete saved screener | One click to re-apply a saved filter set; × to delete |

Fundamentals and technicals (P/E, ROE, RSI, MACD, etc.) are generated by a
seeded pseudo-random function per symbol (`lib/mock-data/screener.ts`), so
values stay stable across reloads but are clearly illustrative — not real
filings. Swap `lib/providers/screener-data.ts` for a real fundamentals API
later without touching any screener UI.

**Note on the database:** after pulling this module, run
`npm run prisma:migrate` again — it adds one new table (`saved_screeners`)
for the "save screener" feature. Everything else (screener universe,
dashboard data) is still in-memory mock data, not stored in Postgres.

## AI Financial Health Score + explanation (company page)

The company page (`/company/[symbol]`) now shows:

1. **A transparent, rule-based Financial Health Score (0–100)** —
   `src/lib/scoring/financial-health.ts`. Every factor (ROE, ROCE,
   debt/equity, revenue growth, profit growth, P/E) shows its exact points,
   weight, and a one-line reason. Nothing is a hidden/black-box number —
   the intent is that anyone can verify a score by hand from the visible
   breakdown. This is illustrative educational scoring on mock
   fundamentals, not a real investment methodology.
2. **An AI-generated plain-English explanation** built from that same
   score breakdown — `src/lib/ai/explain-stock.ts`.
   - If `ANTHROPIC_API_KEY` is set in `.env`, this calls the real Claude
     API and shows a **"Live AI"** badge.
   - If no key is set (or the call fails for any reason — rate limit,
     network, etc.), it falls back to a deterministic template built from
     the same factor data, shown with a **"Template"** badge — so the
     feature always works, with or without a live key.
   - Responses are cached in-memory per symbol for 10 minutes to avoid
     redundant API calls.

To turn on live AI: add a real Anthropic API key to `ANTHROPIC_API_KEY` in
`.env` (get one at https://console.anthropic.com), restart `npm run dev`,
and revisit any company page — the badge switches from "Template" to "Live
AI" automatically, no code changes needed.

## Live NSE stock prices (Twelve Data)

Dashboard, screener, watchlist, and company pages can show **real NSE
price/change/volume** instead of mock data, once configured:

1. Get a free API key at https://twelvedata.com (free tier: 8
   requests/minute, 800/day).
2. Add it to `.env`: `TWELVE_DATA_API_KEY="your-key-here"`.
3. Add a random secret to protect the refresh endpoint from strangers:
   `CRON_SECRET="something-random"` (generate with `openssl rand -hex 16`).
4. Run a refresh manually: `npm run live-data:refresh`.
5. Reload the dashboard — the badge switches from "Illustrative sample
   data" to "Live NSE prices".

**Architecture — why prices are cached, not fetched live per request:**
the free tier's rate limit (8/min) can't support fetching fresh data on
every page load, so a background job refreshes a `live_quotes` table in
Postgres on an interval, and every page reads from that table. This keeps
the app fast and never rate-limited, at the cost of prices being a few
minutes old rather than real-time — a normal, honest tradeoff on a free
API tier.

**What stays mock even with a live key:** market indices (NIFTY/SENSEX —
not available on Twelve Data's free tier), all fundamentals (P/E, ROE,
ROCE, EPS, RSI, MACD — no free fundamentals provider is wired in), and
news. Only price/change/volume for the 20-stock universe go live.

**Keeping the cache fresh automatically:**
- **On Vercel:** `vercel.json` includes a Cron Job hitting the refresh
  endpoint. ⚠️ Vercel's free Hobby plan only allows Cron Jobs to run once
  per day, not every 5 minutes — for more frequent refreshes on the free
  tier, use a free external pinger like https://cron-job.org to hit
  `https://your-app.vercel.app/api/live-data/refresh?secret=YOUR_CRON_SECRET`
  every few minutes instead.
- **Locally:** run `npm run live-data:refresh` manually whenever you want
  fresh prices, or set up a scheduled task/cron on your own machine.

If `TWELVE_DATA_API_KEY` is unset, or the cache is empty/stale, every page
automatically falls back to mock data — nothing breaks either way.

```
  app/

    api/auth/...                              → auth API routes
    api/dashboard/                            → dashboard JSON API

    company/[symbol]/                         → basic company page (mock data)
  components/
    auth/                                     → auth UI (forms, shell, ticker)
    dashboard/                                → dashboard UI (panels, header, widgets)
    ui/                                       → shared primitives (Button, FormField, Toaster)
  lib/
    auth/                                     → jwt, password, cookies, current-user
    providers/market-data.ts                  → data provider seam (mock now, real API later)
    mock-data/stocks.ts                       → illustrative stock/index/news data
    market-types.ts                           → shared data shapes
    validation/                               → Zod schemas
    stores/                                   → Zustand stores (auth, toast, recent searches)
    prisma.ts                                 → Prisma client singleton
    format.ts                                 → number/currency/date formatting
  middleware.ts                               → route protection
prisma/
  schema.prisma                               → User, Session, AuditLog models
  seed.ts                                     → demo data
```

## Security notes

- Passwords hashed with bcrypt (12 rounds), never stored or logged in plain text.
- Login and registration return generic error messages to avoid leaking which emails have accounts.
- JWT secret and all provider API keys are read from environment variables only — never hardcoded.
- Reset tokens are single-use, opaque, and expire after 1 hour.
